# 寫作風格規範

## 核心原則

**段落式描述，層層遞近，禁止清單羅列。**

## 禁止的寫作模式

```markdown
<!-- 禁止：清單羅列 -->

### 模塊列表

- 用户模塊：處理用户相關功能
- 定單模塊：處理訂單相關功能
- 支付模塊：處理支付相關功能

### 依賴關係

| 模塊 | 依賴 | 說明 |
| ---- | ---- | ---- |
| A    | B    | xxx  |
```

## 推薦的寫作模式

```markdown
<!-- 推薦：段落式描述 -->

### 模塊架構設計

系統采用分層模塊化架構，核心業務邏輯圍繞用戶、訂單、支付三大領域展開。
用戶模塊作為系統的入口層，承擔身份認證與權限管理職責，為下遊模塊提供
統一的用戶上下文。訂單模塊位於業務核心層，依賴用戶模塊獲取會話信息，
並協調支付模塊完成交易閉環。

值得注意的是，支付模塊采用策略模式實現多渠道支付，通過接口抽象與
具體支付網關解耦。這一設計使得新增支付渠道時，僅需實現相應策略類，
無需修改核心訂單邏輯，體現了開閉原則的應用。

從依賴方向分析，系統呈現清晰的單向依賴：表現層依賴業務層，業務層
依賴數據層，未發現循環依賴。這一架構特征確保了模塊的獨立可測試性，
同時為後續微服務拆分奠定了基礎。
```

## 寫作策略

### 策略一：主語轉換

將主語從開發者視角轉移到系統/代碼本身：

| 禁止            | 推薦              |
| --------------- | ----------------- |
| 我們設計了...   | 系統采用...       |
| 開發者實現了... | 該模塊通過...     |
| 代碼中使用了... | 架構設計體現了... |

### 策略二：邏輯連接

使用連接詞確保段落遞進：

- **承接**：此外、進一步、在此基礎上
- **轉折**：然而、值得注意的是、不同於
- **因果**：因此、這一設計使得、由此可見
- **總結**：綜上所述、從整體來看、概言之

### 策略三：深度闡釋

每個技術點需包含：

1. **是什麽**：客觀描述技術實現
2. **為什麽**：闡釋設計意圖和考量
3. **影響**：說明對系統的影響和價值

```markdown
<!-- 示例 -->

系統采用依賴注入模式管理組件生命周期（是什麽）。這一選擇源於
對可測試性和松耦合的追求（為什麽）。通過將依賴關系外置於
配置層，各模塊可獨立進行單元測試，同時為運行時替換實現
提供了可能（影響）。
```

## 章節模板

### 架構概述（段落式）

```markdown
## 系統架構概述

{項目名稱}采用{架構模式}架構，整體設計圍繞{核心理念}展開。
從宏觀視角審視，系統可劃分為{N}個主要層次，各層職責明確，
邊界清晰。

{表現層/入口層}作為系統與外部交互的唯一入口，承擔請求解析、
參數校驗、響應封裝等職責。該層通過{框架/技術}實現，遵循
{設計原則}，確保接口的一致性與可維護性。

{業務層}是系統的核心所在，封裝了全部業務邏輯。該層采用
{模式/策略}組織代碼，將覆雜業務拆解為{N}個領域模塊。
值得注意的是，{關鍵設計決策}體現了對{質量屬性}的重視。

{數據層}負責持久化與數據訪問，通過{技術/框架}實現。
該層與業務層通過{接口/抽象}解耦，使得數據源的替換
不影響上層邏輯，體現了依賴倒置原則的應用。
```

### 設計模式分析（段落式）

```markdown
## 設計模式應用

代碼庫中可識別出{模式1}、{模式2}等設計模式的應用，
這些模式的選擇與系統的{核心需求}密切相關。

{模式1}主要應用於{場景/模塊}。具體實現位於
`{文件路徑}`，通過{實現方式}達成{目標}。
這一模式的引入有效解決了{問題}，使得{效果}。

在{另一場景}中，系統采用{模式2}應對{挑戰}。
不同於{模式1}的{特點}，{模式2}更側重於{關注點}。
從`{文件路徑}`的實現可以看出，設計者通過
{具體實現}實現了{目標}。

綜合來看，模式的選擇體現了對{原則}的遵循，
為系統的{質量屬性}提供了有力支撐。
```

### 算法流程分析（段落式）

```markdown
## 核心算法設計

{算法名稱}是系統處理{業務場景}的核心邏輯，
其實現位於`{文件路徑}`。

從算法流程來看，整體可分為{N}個階段。首先，
{第一階段描述}，這一步驟的目的在於{目的}。
隨後，算法進入{第二階段}，通過{方法}實現{目標}。
最終，{結果處理}完成整個處理流程。

在覆雜度方面，該算法的時間覆雜度為{O(x)}，
空間覆雜度為{O(y)}。這一覆雜度特征源於
{原因}，在{數據規模}場景下表現良好。

值得關注的是，{算法名稱}采用了{優化策略}，
相較於樸素實現，{具體優化點}。這一設計決策
使得{性能提升/效果}。
```
